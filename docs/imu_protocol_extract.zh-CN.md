# IMU 协议提取

**语言 / Language：** [English](./imu_protocol_extract.md) | **简体中文**

## 模块身份

在本项目中，`ATK-IMU901` 与 `ATK-MS901M` 视为同一模块族。实现统一依据 `ATK-MS901M` 的协议文档和示例代码。

## 传输方式

- 物理链路：UART
- 连接接口：`UART0`
- 默认波特率：`115200`

## 帧结构

- 上传帧头：`0x55 0x55`
- 寄存器或 ACK 帧头：`0x55 0xAF`
- 校验方式：校验字段之前所有字节的和

## 上传帧 ID

| ID | 含义 |
|---|---|
| `0x01` | 姿态 |
| `0x02` | 四元数 |
| `0x03` | 陀螺仪 + 加速度计 |
| `0x04` | 磁力计 |
| `0x05` | 气压计 |
| `0x06` | 端口状态 |

## 相关寄存器 ID

| ID | 含义 |
|---|---|
| `0x07` | 串口波特率 |
| `0x08` | 回传内容集合 |
| `0x0A` | 回传频率 |
| `0x0B` | 算法选择 |
| `0x0C` | 安装方向 |

## 已确认回传频率表

| 代码 | 频率 |
|---|---|
| `0x00` | `250 Hz` |
| `0x01` | `200 Hz` |
| `0x02` | `125 Hz` |
| `0x03` | `100 Hz` |
| `0x04` | `50 Hz` |
| `0x05` | `20 Hz` |
| `0x06` | `10 Hz` |
| `0x07` | `5 Hz` |
| `0x08` | `2 Hz` |
| `0x09` | `1 Hz` |

## 固件模式拆分

### 模式 A：原始传感器模式

- 默认回传内容：`gyro + acc`
- 可选回传内容：`mag`
- 默认关闭：`baro`、`attitude`、`quaternion`
- ESP32 侧职责：打时间戳、坐标映射、陀螺零偏校准、四元数解算、欧拉角输出和健康状态输出

### 模式 B：直出姿态模式

- 默认回传内容：`attitude + quaternion`
- 原始帧默认关闭
- ESP32 侧职责：帧解析、坐标映射、有效性过滤、超时检测、离群值抑制和健康状态输出

## 统一输出契约

上层统一消费同一种 `imu_sample_t`：

- `timestamp_us`
- `gyro_xyz_dps`
- `acc_xyz_g`
- `quat_wxyz`
- `roll_pitch_yaw_deg`
- `health`
- `update_age_us`

## 运行时设计约束

模块主动上传频率最高只有 `250 Hz`，因此控制架构必须采用：

- `1 kHz` 电机或输出调度
- estimator 仅在 fresh IMU sample 到来时更新
- rate PID 仅在 fresh IMU sample 到来时更新
- 更低频率的 angle PID

固件不能通过反复处理旧 IMU 数据来伪造 `1 kHz` 传感器更新。
